# OpenProvenceプロジェクト ドキュメント作成完了レポート

## 概要

OpenProvenceプロジェクト（`../open_provence/`）に必要なドキュメントとプロジェクト構造を整備しました。

## 作成したドキュメント

### 1. CLAUDE.md
**目的**: Claude Codeがプロジェクトで作業する際のガイドライン

**主な内容**:
- プロジェクト概要（query-dependent text pruning）
- 開発環境設定（UV使用）
- アーキテクチャ概要（OpenProvenceEncoder、モード説明）
- コーディングガイドライン
- パフォーマンスベースライン（F2.5: 0.8483±0.01）
- トラブルシューティング

### 2. docs/spec.md
**目的**: 技術仕様の詳細な記述

**主な内容**:
- コアアーキテクチャ（モデル構造、訓練アーキテクチャ）
- アルゴリズム詳細（pruning決定プロセス、文境界検出）
- データ構造（入力/出力フォーマット）
- API リファレンス
- 設定ファイル形式
- 互換性情報
- sentence-transformersからの移行ガイド

### 3. README.md
**目的**: プロジェクトの概要と使用方法

**主な内容**:
- インストール手順
- クイックスタート（訓練と推論）
- モードの説明（pruning-only vs reranking+pruning）
- データフォーマット
- 期待される性能指標
- 使用例とコード snippets

### 4. CONTRIBUTING.md
**目的**: 貢献者向けガイドライン

**主な内容**:
- 開発環境のセットアップ
- 開発ワークフロー
- コードスタイルガイドライン
- テストガイドライン
- PR提出プロセス
- イシュー報告テンプレート

### 5. examples/basic_usage.py
**目的**: 基本的な使用例の提供

**主な内容**:
- モデルのロードと使用
- バッチ処理の例
- 評価データフォーマット
- 閾値の使い方
- モードの違いの説明

### 6. .gitignore
**目的**: バージョン管理から除外するファイルの指定

**主な内容**:
- Python標準の除外パターン
- UV/仮想環境関連
- モデルファイル（大きすぎるため）
- 出力ディレクトリ
- 一時ファイル

## プロジェクト構造

```
open_provence/
├── .gitignore              # Git除外設定
├── CLAUDE.md               # AI assistant向けガイド
├── CONTRIBUTING.md         # 貢献者ガイド
├── README.md               # プロジェクト概要
├── pyproject.toml          # プロジェクト設定（既存）
├── docs/
│   └── spec.md            # 技術仕様書
├── examples/
│   └── basic_usage.py     # 使用例
├── configs/               # 訓練設定（既存）
├── scripts/               # 実行スクリプト（既存）
├── open_provence/         # ソースコード（既存）
└── tests/                 # テスト（既存）
```
