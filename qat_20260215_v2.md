## Linear Staged Warmup (What Was Implemented)

- I implemented a linear step-based warmup schedule in my local branch.
- Base QAT weights were `[1.0, 1.0, 0.5]` for `[float32, int8, binary]`.
- Warmup steps were set to `float32=0`, `int8=200`, `binary=1800`.
- Effective weight grows linearly by step until each target weight is reached.

## Investigation Findings (Current)

- The cache-fixed behavior generally improved robustness versus plain PR implementation, especially for quantized precisions.
- Warmup behavior was sensitive to dataset size and batch size.
- In `90k`, binary warmup (`1800`) does not finish (`steps=1407` for `bs=64`, `steps=704` for `bs=128`), so binary weight never reaches its target by training end.
- At this stage, a stable quantization recipe that is robust across data scale and batch size has not been identified yet.
