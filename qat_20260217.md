# QAT Ablation Summary (2026-02-17)

## Scope
This note summarizes today's QAT ablations focused on:
- `phaseN0` to `phaseN10` (seed=42, 300k samples)
- baseline seed sweep (`seed=40,41,43`) under the same baseline recipe

## Common Setup
- Model: `microsoft/mpnet-base`
- Train dataset: `sentence-transformers/gooaq` (`train` split)
- Train samples: `300000`
- Batch size: `128`
- Epochs: `1.0`
- Learning rate: `2e-5`
- Eval benchmark: NanoBEIR (all 13 datasets)

- Eval mode: `evaluator`
- Eval int8 range: `quantile=0.995`
- Query quantization at eval: `false` (`--no-eval-quantize-queries`)
- Rescore: `false` (`--no-eval-rescore`)
- Offline flags: `HF_HUB_OFFLINE=1 HF_DATASETS_OFFLINE=1`
- Main metric in tables: mean NanoBEIR `nDCG@10` (`float32 / int8 / binary`)

## Current Readout
- Some QAT variants (N1/N6/N9/N10) are close to baseline and sometimes slightly better on one precision.
- Across all three precisions together, no robust and consistent gain is observed over baseline.
- Seed sensitivity is non-trivial; one baseline seed run (`seed=43`) is the best single run in this batch.

## Practical Conclusion (Current)
At this stage and this setting (`300k, bs=128, full NanoBEIR eval`), baseline MNRL appears strong enough and QAT recipe improvements are not yet stable.
