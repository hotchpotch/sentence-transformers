## Qdrant-Derived Next Steps (No Rerank Path)

The following are prioritized for next iterations under the current no-rerank policy:

1. Robust int8 calibration path in evaluator and training alignment
- Keep `minmax` as control, test `quantile` ranges systematically (e.g., `0.99`, `0.995`, `0.999`).
- Add train-side range smoothing ablation (`train_int8_range_momentum`) to reduce calibration variance.
- Goal: recover int8 without degrading float32.

2. Asymmetric query strategy without rerank
- Keep corpus quantized while evaluating float query vs quantized query under the same retrieval path.
- Use this as a controlled binary-int8 tradeoff axis (no oversampling/rescoring).

3. Binary objective shaping (train-side)
- Current binary loss behavior is unstable across scales.
- Introduce weight/warmup schedules that are step-ratio aware (not fixed absolute steps only), then re-check at 300k and 1M.

4. Distance-consistency regularization (train-side)
- Add a light consistency term between float32 and quantized similarities to reduce ranking drift.
- Evaluate whether this improves binary and int8 jointly while preserving float32.

5. Two-stage validation ladder
- Fast filter: `300k, bs=128` (full NanoBEIR, offline).
- Promotion gate: only candidates that beat control on at least int8+binary while staying close on float32 move to `1M`.

## Notes from Qdrant Source Review

- Qdrant explicitly separates quantized retrieval controls (`oversampling`, `rescore`) from quantization construction and defaults binary rescoring to true.
- Scalar int8 path supports quantile clipping and distance-specific multiplier restoration.
- Binary path supports asymmetric query encodings (`Scalar4bits`, `Scalar8bits`), which indicates query-side precision is a strong lever for binary quality.
- For our current constraint (no rerank), the most transferable ideas are:
  - robust calibration (quantile and smoothing),
  - asymmetric query handling,
  - train-side consistency to reduce quantized ranking drift.
