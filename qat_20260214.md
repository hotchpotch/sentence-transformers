## Implementation Differences (Baseline / trainB-control / step4a / step4b / step5)

Only these training knobs differ:
- `train_loss`
- `train_binary_mode`
- `train_use_int8_range_state`
- `train_quantization_warmup_steps`
- `train_precision_warmup_steps`

Detailed table:

| Variant | train_loss | train_binary_mode | train_use_int8_range_state | train_quantization_warmup_steps | train_precision_warmup_steps |
| --- | --- | --- | --- | ---: | --- |
| baseline | mnrl | unsigned | true | 200 | null |
| trainB-control | qat | signed | false | 0 | null |
| step4a | qat | signed | false | 0 | [0, 200, 1200] |
| step4b | qat | signed | true | 0 | [0, 200, 1200] |
| step5 | qat | signed | true | 0 | [0, 200, 1200] |

Important behavior notes:
- In baseline (`train_loss=mnrl`), QAT-specific args are effectively inactive in the loss path.
  - Reference: `examples/sentence_transformer/training/quantization/train_qat_gooaq_ablation.py`
- `step4a` adds per-precision warmup schedule on top of trainB-control.
  - Reference: `sentence_transformers/losses/QuantizationAwareLoss.py` (`_scaled_weight`)
- `step4b` enables int8 EMA range-state on top of step4a.
  - Reference: `sentence_transformers/losses/QuantizationAwareLoss.py` (EMA range update path)
- `step5` keeps step4b hyperparameters but includes cache isolation fix in forward cache usage.
  - Reference: `sentence_transformers/losses/QuantizationAwareLoss.py` (`ForwardDecorator.__call__`, cached output copy)
